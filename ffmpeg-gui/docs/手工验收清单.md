# 手工验收清单

本文用于稳定基线 / Release Candidate 验收。目标是确认核心处理链路可靠，而不是继续增加功能。

## 自动验证

在仓库根目录执行：

```bash
.venv/bin/python -m compileall desktop shared tests
.venv/bin/python -m pytest
RUN_FFMPEG_GUI_SMOKE=1 .venv/bin/python -m pytest tests/integration
```

验收要求：

- `compileall` 通过。
- 单元测试和默认集成测试通过。
- 真实 ffmpeg smoke 通过；如果本机没有 `ffmpeg` / `ffprobe`，记录为环境跳过，不视为产品通过。

## 手工流程

### 1. 单文件处理

- 选择一个带音频的视频文件。
- 确认任务表当前行在右侧媒体预览中显示输入文件，可播放、暂停和拖动时间。
- 在预览中点击“设为开始”“设为结束”，确认处理范围字段和命令预览刷新。
- 切到 `thumbnail` 动作，在预览中点击“设为封面时间”，确认封面时间参数刷新。
- 分别运行 `convert`、`compress`、`gif fast`。
- 确认命令预览随参数变化刷新。
- 确认任务表状态从 running 到 succeeded。
- 任务成功后切换媒体预览到“输出”，确认输出文件可预览；如果本机解码器不支持该文件，只记录预览不可用但任务成功。
- 确认输出文件存在、可打开、可复制路径、可打开输出目录。
- 确认 `data/logs/` 中写入对应任务日志。

### 2. 批处理

- 选择 2 个以上视频文件进入批处理。
- 运行 `convert` 或 `resize_compress`。
- 确认总进度、当前项进度、待处理数量和按钮状态正确。
- 确认最近批次统计显示成功、失败、取消和已打包数量。
- 使用“复制最近批次路径”“打开最近批次目录”“定位最近批次结果”。

### 3. Batch + Stack

- 建立 Stack：`resize_compress` -> `crop` -> `fade`。
- 用两个不同分辨率文件跑批处理。
- 确认每个文件按自身媒体信息构建命令。
- 故意设置一个会超出小分辨率文件的 crop 区域。
- 确认超出文件被标记 failed，其他文件继续处理，不生成坏命令。

### 4. Palette GIF

- 选择 `gif` 操作，质量选 `palette`。
- 设置 `fps`、`width`、`start_seconds`、`end_seconds`。
- 确认输出为真实 GIF 且大小大于 0。
- 任务成功后检查 `data/temp/` 没有残留 `*_palette.png`。
- 再运行一次后中途取消，确认 palette 临时文件仍被清理。

### 5. ZIP All

- 在一次批处理中制造至少 1 个成功输出和 1 个失败或取消任务。
- 点击“打包成功结果”。
- 确认 ZIP 写入当前输出目录，命名类似 `ffmpeg-gui-batch-YYYYMMDD-HHMMSS.zip`。
- 确认状态栏显示“已打包 N 个，跳过 M 个”。
- 打开 ZIP，确认只包含最近批次成功输出，不包含源文件、不包含失败/取消/历史批次。
- 用重复文件名输出验证 ZIP 内文件名自动加序号。

### 6. 取消任务

- 启动一个耗时操作，例如大文件 `compress` 或 `palette` GIF。
- 点击取消当前任务。
- 确认任务状态为 cancelled。
- 确认没有悬挂 `ffmpeg` 进程。
- 确认临时 palette 文件被清理。
- 确认日志中记录取消过程。

### 7. 关闭窗口

- 启动一个耗时 ffmpeg 任务后直接关闭窗口。
- 重新检查进程列表，确认没有残留 `ffmpeg` / `ffprobe` / `caffeinate`。
- 再启动 ZIP 打包后关闭窗口，确认 ZIP worker 被取消，不留下半成品 ZIP。

### 8. 防睡眠开关

- macOS 下保持默认开启，启动一个长任务。
- 确认状态栏出现“长任务防睡眠已启用”。
- 任务运行时进程列表中可看到 `caffeinate -dimsu`。
- 任务结束、取消或关闭窗口后确认 `caffeinate` 退出。
- 在设置里关闭“长任务期间防止系统睡眠”，再次运行任务，确认不会启动 `caffeinate`。
- 非 macOS 平台只确认任务正常运行，不要求出现 `caffeinate`。

## 清理检查

自动测试或手工验收结束后检查：

```bash
find data/temp -maxdepth 1 -name '*_palette.png' -print
find data/temp -maxdepth 1 -type f -print
ps -ef
```

验收要求：

- `data/temp/` 不残留 palette PNG。
- 取消或失败后不残留半成品 ZIP。
- 关闭窗口后不残留本应用启动的 `ffmpeg`、ZIP worker 线程或 `caffeinate`。
- `data/logs/` 可以保留任务日志；`data/outputs/` 可以保留用户输出。
