# TODO 07: GUI 预览与交互补齐

## 目标

补齐上游非模型体验中对本机 GUI 有实际价值的部分：输入预览、命令预览、输出估算、默认值填充和更清晰的状态反馈。不做 Web/PWA 外观复刻。

## Checklist

- [ ] 输入预览：
  - [ ] 对视频/音频显示基础媒体信息。
  - [ ] 如使用 QtMultimedia 成本可控，则增加简单播放预览。
  - [ ] 如果 QtMultimedia 依赖或平台表现不稳定，先保留信息预览，不阻塞功能复现。
- [ ] Trim 交互：
  - [ ] 保留当前 start/end 秒数。
  - [ ] 有 duration 时可增加简单 slider。
  - [ ] 不做复杂时间线、波形或逐帧裁剪 UI。
- [ ] 默认值填充：
  - [ ] crop 默认使用视频尺寸内的合理区域。
  - [ ] thumbnail 默认时间点为中点。
  - [ ] resize 默认读取源尺寸。
  - [ ] pad 默认 `16:9 black`。
- [ ] 命令预览：
  - [ ] 每次参数变化后展示将执行的参数数组。
  - [ ] 预览必须来自 runtime builder，不手写第二套命令生成。
  - [ ] 错误参数显示 validation message。
- [ ] 输出大小估算：
  - [ ] 显示保守估算或“不确定”。
  - [ ] 不做复杂码率预测模型。
- [ ] 结果入口：
  - [ ] 打开输出文件。
  - [ ] 打开输出目录。
  - [ ] 复制输出路径。
  - [ ] 清理本次任务日志。
- [ ] 错误可读性：
  - [ ] ffmpeg 缺失、ffprobe 缺失、输入缺失、输出不可写、参数非法分别显示明确文案。

## 反目标

- 不像素级复刻上游网页 UI。
- 不引入 WebView。
- 不引入图标库、主题系统或复杂皮肤。
- 不做拖拽裁剪编辑器。
- 不做实时输出预览或边转码边播放。

## 验收

- 常用路径下用户能在一个窗口内完成：选文件、选操作、看参数、看命令、运行、看进度、打开结果。
- 375px 这类移动 Web 适配不是 GUI 验收目标；只要求桌面窗口在最小尺寸下无明显遮挡。
- offscreen 启动检查通过。
