## 批处理基线

- 批处理只运行 `BATCH_SUPPORTED_OPERATIONS` 中的单文件操作，不把多输入操作自动展开。
- 批处理队列按文件逐个构建命令，输出写入当前输出目录。
- `fade` 的淡出时长按每个文件自己的 `duration_seconds` 注入。
- `crop` 在批处理和 Stack 批处理中按每个文件自己的视频分辨率预检；超出分辨率的文件标记失败，不静默生成坏命令。
- 批处理完成后保留最近批次结果状态，用于复制成功输出路径、打开输出目录、定位任务表行和 ZIP 打包。

## Stack 基线

- Stack 只支持可串联的 filter 类操作：`resize_compress`、`crop`、`rotate`、`adjust`、`denoise`、`sharpen_blur`、`pad`、`volume`、`speed`、`fade`。
- 单个 Stack 最多 6 个步骤。
- trim 只允许出现在 Stack 第一个步骤。
- Stack 批处理与普通批处理共享逐文件媒体上下文，支持逐文件 `crop` 预检和 `fade_out` 时长计算。

## GIF 质量

- `gif` 操作支持两种质量模式：
  - `fast`：单阶段 `fps,scale` 路径，启动快。
  - `palette`：两阶段 `palettegen/paletteuse` 路径，临时 palette 写入 `data/temp/`，任务结束、失败或取消后清理。
- 两种模式都支持 `fps`、`width`、`start_seconds`、`end_seconds`。
- Stack 模式提供 Stack 级输出选项：默认跟随最后一步，也可以选择 `gif` 并配置 `fast/palette`、`fps`、`width`。
- 常规单操作仍只有 `gif` 操作提供 palette 质量策略；其他视频操作的常规输出格式不扩展为 GIF。

## 结果管理

- 最近批次状态记录成功、失败、取消和已打包数量。
- “打包成功结果”只收集最近批次中 `succeeded` 且输出文件仍存在的结果。
- ZIP 写入当前输出目录，命名格式为 `ffmpeg-gui-batch-YYYYMMDD-HHMMSS.zip`。
- 失败、取消、无输出、文件已删除的任务会跳过；不压缩源文件，不自动混入历史批次。
- ZIP 内重复文件名会自动加序号避免覆盖。

## 内嵌媒体预览

- 任务表当前行会驱动右侧媒体预览面板。
- 预览使用 PySide6 / Qt Multimedia 的本机控件，不引入 WebView、浏览器播放器或 ffmpeg.wasm。
- 支持输入预览；任务成功且输出文件存在时，可切换到输出预览。
- 支持播放、暂停、停止、seek 和当前时间显示。
- 可把当前时间写回通用 `start_seconds` / `end_seconds` 处理范围。
- 当动作是 `thumbnail` 时，可把当前时间写回 `timestamp_seconds`。
- 当前不做逐文件独立编辑参数；批处理仍使用当前参数表单作为统一参数。

## 长任务防睡眠

- 设置中默认开启“长任务期间防止系统睡眠”。
- 只在 ffmpeg 任务或 ZIP 打包运行期间启用，普通探测、表单编辑和空闲状态不启用。
- macOS 使用 `caffeinate -dimsu`。
- Windows 使用 `SetThreadExecutionState`。
- Linux 使用 `systemd-inhibit --what=sleep:idle` 持有防睡眠锁。
- 平台能力或命令启动失败只在状态栏提示，不阻塞任务。
- 任务成功、失败、取消或关闭窗口后释放防睡眠状态或停止防睡眠进程。

## 明确不做

- 不接入 Auto-Caption、Whisper、Transformers.js 或任何自动字幕模型。
- 不接入 `ffmpeg.wasm`。
- 不做 PWA、service worker、浏览器离线安装、COOP/COEP 或浏览器 Wake Lock。
- 不做 WebUI、FastAPI、本地 HTTP 上传、Electron、Tauri、React、Vue 或 QML。
- 当前桌面版不做 Web/PWA 播放器、逐帧精修时间线、可视化裁剪拖拽器或滤镜实时预览。
